Recently, I ran into an issue with a jQuery time picker plugin not firing my custom validator `onChange` event in ASP.NET.

The issue seemed to be related to the `ControlToValidate` setting.

Based on <a href="http://stackoverflow.com/questions/8649565/do-not-fire-custom-validate-on-onchange-event-of-a-textbox" target="_blank" rel="nofollow noopener noreferrer">this Stack Overflow question</a>, using a `ValidationGroup` in the custom validator resolved the problem.
